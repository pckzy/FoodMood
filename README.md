## Problem Statement

"วันนี้กินอะไรดีนะ?" เป็นประโยคยอดฮิตที่สะท้อนถึงปัญหา Choice Overload หรือการมีตัวเลือกมาก
เกินไปจนตัดสินใจไม่ได้ นอกจากนี้ งานวิจัยทางจิตวิทยาระบุว่าสภาวะอารมณ์ (Mood) และสภาพ
แวดล้อมภายนอก (Weather) มีผลอย่างมากต่อความต้องการเมนูอาหารและรสชาติในช่วงเวลานั้นๆ

แอปพลิเคชัน **FoodMood** จึงถูกพัฒนาขึ้นเพื่อช่วยให้ผู้ใช้ลดภาระในการตัดสินใจ
ด้วยระบบการแนะนำอาหารที่ชาญฉลาด โดยใช้ปัจจัยรอบตัวมาเป็นตัวขับเคลื่อนการค้นหา
พร้อมมอบประสบการณ์การใช้งานที่แปลกใหม่ผ่านระบบการปัดการ์ด (Swipe UI)
ที่ช่วยให้การเลือกเมนูอาหารกลายเป็นเรื่องที่ง่าย

---

## Target Users

- กลุ่มคนที่ตัดสินใจเลือกเมนูอาหารด้วยตัวเองไม่ได้
- ผู้ที่ต้องการค้นหาเมนูอาหารใหม่ๆ ตามความรู้สึก
- ผู้ที่อยากลองแอปพลิเคชั่นใหม่ที่แตกต่าง

---

## MVP Features

1. Recommendation Logic Engine: ระบบคัดกรองข้อมูลจาก Database เพื่อดึงเมนูอาหารที่ตรงกับอารมณ์และสภาพอากาศที่ผู้ใช้เลือกมาแสดง
2. Tinder-Style Swipe UI: ระบบแสดงการ์ดเมนูอาหารที่ผู้ใช้สามารถปัดซ้าย (ไม่สนใจ) หรือปัดขวา (สนใจ) ได้
3. Image Tap Navigation: ระบบแตะพื้นที่ด้านซ้าย/ขวาของรูปเพื่อสลับดูรูปภาพอื่นๆ ของเมนูเดียวกัน
4. Information Overlay: ปุ่ม (i) หรือปุ่ม "View Detail" บนการ์ดเมื่อกดที่การ์ด จะแสดงรายละเอียดของเมนูนั้นๆ
5. Match History: ระบบเก็บรายการเมนูที่ "ปัดขวา" ไว้ดูย้อนหลัง
6. Empty State & Reset Logic: ระบบจัดการเมื่อการ์ดหมดเพื่อให้ใช้งานต่อได้
7. Session & State Management: ระบบมีการจัดการสถานะการปัดการ์ดแบบชั่วคราว โดยเมนูที่ถูกปฏิเสธ (ปัดซ้าย) จะถูกซ่อนไว้เฉพาะในช่วงเวลามื้ออาหารนั้นๆ และจะถูกนำกลับมาแนะนำใหม่เมื่อเข้าสู่ช่วงเวลาถัดไป หรือเมื่อผู้ใช้กดรีเซ็ตระบบ เพื่อรองรับความต้องการอาหารที่เปลี่ยนแปลงไปตามช่วงเวลาของวัน

## Nice-to-Have Features (Future Work)

1. Blacklist System: ระบบกรองอาหารขั้นสูงที่ผู้ใช้สามารถกด Block บางเมนูไว้ เพื่อให้ Recommendation Engine ฉลาดต่อผู้ใช้มากขึ้น
2. Recommend of the Day: ฟีเจอร์ที่ช่วยลดขั้นตอนการตัดสินใจของผู้ใช้งาน โดยการนำเสนอเมนูอาหารที่น่าสนใจประจำวันแบบสุ่ม สร้างความแปลกใหม่ในการใช้งาน
3. Mood & Food Analytics: ระบบสรุปสถิติรายเดือนว่า เดือนที่ผ่านมาคุณมีอารมณ์แบบไหนบ่อยที่สุด และ เมนูไหนคืออาหารที่คุณเลือกบ่อยที่สุดในเดือนนั้นๆ
4. Multi-language (TH / EN): ขยายกลุ่มผู้ใช้งานให้ครอบคลุมทั้งชาวไทยและชาวต่างชาติ
5. Dark-Light Theme: มอบทางเลือกในการตั้งค่าที่เหมาะสมกับสภาพแสงในแต่ละช่วงเวลา

## Non-Functional Requirements

- **Performance**  
  การแสดงผล Card Swiping ต้องมีความเร็วไม่ต่ำกว่า 60 FPS เพื่อให้รู้สึกลื่นไหล

- **Usability**  
  UI ต้องออกแบบมาให้ใช้งานได้ด้วย "มือเดียว" เนืองจากผู้ใช้อาจใช้งานขณะเดินทาง

- **Availability**  
  ข้อมูลรูปภาพอาหารต้องมีการทำ Caching เพื่อให้โหลดได้เร็วแม้ในจุดที่สัญญาณอินเทอร์เน็ตไม่ดี

- **Responsiveness**  
  UI ต้องรองรับการแสดงผลที่สมบูรณ์ในหน้าจอมือถือหลายขนาด  

- **Zero-Cost Infrastructure**  
  ระบบหลังบ้าน (Backend) ต้องรันบน Free Tier ของ Supabase โดยไม่มีค่าใช้จ่ายรายเดือน

- **Clean Architecture**  
  จัดระเบียบโครงสร้างซอฟต์แวร์ให้เป็นระเบียบ โดยแยก Business Logic ออกจาก Framework และ UI อย่างชัดเจน
